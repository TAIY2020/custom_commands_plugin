# 自定义命令插件 (CustomCommands Plugin)

**一个允许你通过聊天命令动态添加、删除、列出和触发自定义回复的强大插件。**

这个插件赋予了麦麦一个"可编程"的记忆。管理员可以在聊天中随时"教"麦麦新的命令和回复，而无需重启或修改任何代码。所有添加的命令都会被永久保存。

> 本版本 (v2.1.0) 基于 **MaiBot SDK v2** 重写，使用 `@Command` 装饰器和 `PluginConfigBase` 强类型配置模型，支持配置热重载和 Dashboard UI 配置。

## ✨ 功能特性

- **动态管理**: 无需重启，通过聊天命令即可实时添加、删除和查看自定义命令。
- **权限控制**: 仅限指定管理员可以添加或删除命令，确保安全可控。
- **多格式支持**: 回复内容可以是纯文本，也可以是图片。
- **路径安全**: 内置路径穿越检测，添加命令时即拒绝非法路径，触发时二次校验，防止恶意文件名访问图片目录以外的文件。
- **图片大小限制**: 自动检测图片文件大小，超过 10MB 的图片将被拒绝发送，防止内存溢出。
- **输入校验**: 触发词最多 50 字符，回复内容最多 2000 字符，每个作用域最多 500 条命令。
- **并发安全**: 所有写操作通过异步锁保护，防止多人同时操作导致数据竞争。
- **异步非阻塞**: 数据保存使用异步写入，不阻塞事件循环，高并发场景下也不会卡顿。
- **原子写入**: 使用"临时文件 + 原子重命名"模式保存数据，防止崩溃导致数据损坏。
- **自包含资源**: 图片等资源默认存放于插件自己的目录中，方便管理和分发。
- **持久化存储**: 所有自定义命令都会自动保存到插件目录下的 `custom_commands.json` 文件中，麦麦重启后依然有效。
- **群组隔离与共享**: 支持不同群聊使用独立的命令库，或通过映射让几个特定的群共享一套命令。
- **继承机制**: 即使开启了隔离，群聊依然可以使用全局通用的命令，实现"公共+私有"的混合模式。
- **配置热重载**: 通过 Dashboard 修改配置后无需重启，插件会自动应用新配置。

## 🚀 快速开始

### 1. 安装

下载 `custom_commands_plugin` 文件夹放入麦麦主程序的 `plugins` 目录下，然后重启主程序即可完成插件的注册和加载。

### 2. 环境要求

- **MaiBot 主程序**: v1.0.0+
- **MaiBot SDK**: v2.0.0+

### 3. 配置

首次启动麦麦后，插件会在其目录下自动生成 `config.toml` (配置文件) 和 `custom_commands.json` (数据文件)。前者是关于插件的一些基础配置，后者是插件自定义命令的存储文件。同时，也会根据配置自动创建存放图片的目录。

你也可以通过 **Dashboard** 在线修改配置，修改后会自动热重载生效。

**默认配置示例**:

```toml
[plugin]
name = "custom_commands_plugin"
version = "2.1.0"
config_version = "2.1.0"
enabled = true

[settings]
command_prefix = "."
admin_user_ids = []
image_directory = "plugins/custom_commands_plugin/images"
enable_group_isolation = false

[settings.group_scopes]
# 群组作用域映射，键为作用域名称，值为该作用域下的群号列表
# 示例: 游戏群 = ["123456", "654321"]
```

**⚠️ 重要安全提示**:

- **`command_prefix`**: 定义了所有命令的触发前缀，默认为 `.`。
- **`admin_user_ids`**: 这是一个**核心安全设置**。默认值为一个**空列表 `[]`**，这意味着**默认情况下，任何人都没有权限**添加或删除命令。**您必须手动编辑配置文件**（或通过 Dashboard），将管理员的 QQ 号添加到这个列表中，才能使用管理员命令。
- **`image_directory`**: 指定了存放回复图片的文件夹路径，相对于主程序根目录。默认路径 `plugins/custom_commands_plugin/images` 会被自动创建。您可以将图片放入此文件夹，然后通过文件名来调用。
- **`enable_group_isolation`**: 控制群组隔离模式。默认为 `false`，表示所有群聊共享同一套全局命令。若设置为 `true`，则每个群聊将拥有独立的命令库，但依然可以调用全局命令。
- **`group_scopes`**: 允许您将多个群号归入同一个作用域名称，实现这些群共享一套命令。键为作用域名称，值为群号列表。例：在 `[settings.group_scopes]` 段下添加 `游戏组 = ["111111", "222222"]`，则这两个群将共享同一套命令。

## 📖 使用指南

假设你的命令前缀 `command_prefix` 设置为 `.`。

### 👨‍💼 管理员命令 (需要配置权限)

#### 添加命令

使用 `.问：...答：...` 的格式来教麦麦新的回复。

- **格式**: `.问：<触发词>答：<回复内容>`
- **限制**: 触发词最多 50 字符，回复内容最多 2000 字符，每个作用域最多 500 条命令。
- **示例 (文本回复)**:

  ```text
  .问：你好答：你好啊！很高兴见到你！😊
  ```

- **示例 (图片回复)**:
  将图片（例如 `hello.png`）放入您在配置文件中指定的 `image_directory` 目录（默认为该插件目录下的 `images` 文件夹），然后使用文件名作为回复内容。

  ```text
  .问：打个招呼答：hello.png
  ```

- **作用域说明**:
  - 如果未开启隔离且未配置映射，命令将添加到**全局**。
  - 如果开启了隔离，命令将添加到**当前群**。
  - 如果当前群配置了映射（如 `game_group`），命令将添加到**分组**。

#### 删除命令

使用 `.删：...` 的格式来让麦麦忘记一个命令。

- **格式**: `.删：<触发词>`
- **示例**:

  ```text
  .删：你好
  ```

- **注意**: 只能删除当前作用域下的命令。如果尝试删除一个全局命令，插件会给出提示并引导你使用 `删全局` 命令。

#### 删除全局命令

在任何群聊中直接删除全局作用域的命令，无需切换隔离模式。

- **格式**: `.删全局：<触发词>`
- **示例**:

  ```text
  .删全局：你好
  ```

### 🙋‍♀️ 通用命令 (任何人可用)

#### 列出所有命令

查看当前所有可用的自定义命令。为了避免自定义命令过多刷屏，麦麦会以一个**合并转发消息卡片**来展示列表。

- **格式**: `.列表`

#### 触发命令

使用 `前缀` + `触发词` 来获取回复。

- **格式**: `<前缀><触发词>`
- **示例**:

  ```text
  .你好
  ```

- **麦麦回复**:

  ```text
  你好啊！很高兴见到你！😊
  ```

## 🧩 进阶功能：群组隔离与共享

### 1. 全局共享模式 (默认)

- **配置**: `enable_group_isolation = false`, `[settings.group_scopes]` 为空
- **效果**: 所有群聊共用同一套命令库。你在 A 群添加的命令，所有群也能用。

### 2. 独立隔离模式

- **配置**: `enable_group_isolation = true`
- **效果**:
  - 在 A 群添加的命令，只有 A 群能触发（数据存储在以 A 群号命名的作用域中）。
  - 继承特性: A 群依然可以触发之前添加在"全局"的命令。如果 A 群添加了一个和全局命令同名的命令，A 群的命令会覆盖全局命令。

### 3. 自定义分组模式 (部分共享)

- **配置**: 在 `[settings.group_scopes]` 下添加 `游戏组 = ["群号A", "群号B"]`
- **效果**:
  - 群 A 和群 B 被绑定到了同一个作用域 `游戏组`。
  - 在群 A 添加的命令，群 B 立马就能用。
  - 群 C（未绑定）依然遵循全局模式或独立模式（取决于 `enable_group_isolation`）。

## 🔄 从 v1.x 升级

v2.1.0 是基于 MaiBot SDK v2 的完全重写版本，主要变化如下：

| 项目 | v1.x (旧版) | v2.1.0 (新版) |
|------|-------------|---------------|
| SDK 依赖 | `src.plugin_system` (内置插件系统) | `maibot_sdk` v2 |
| 命令注册 | `BaseCommand` 子类 + `get_plugin_components()` | `@Command` 装饰器 |
| 配置管理 | `config_schema` 字典 + `ConfigField` | `PluginConfigBase` 强类型模型 (Pydantic) |
| 配置热重载 | 不支持 | 支持 `on_config_update` 回调 |
| 思考拦截 | 自定义 `ThinkingInterceptor` 单例 + `EventHandler` | SDK 内置机制，命令返回值控制 |
| 图片安全 | 无路径校验 | 添加时即校验路径穿越 + 触发时二次校验 + 文件大小限制 (10MB) |
| 数据安全 | 直接写入文件，无并发保护 | 异步锁 + 异步非阻塞写入 + 原子写入 + 深层数据校验 |
| 输入校验 | 无限制 | 触发词/回复长度限制 + 命令数量上限 |
| 删除全局命令 | 需切换隔离模式 | 新增 `删全局：` 命令，任何群可直接删除 |
| 清单文件 | `manifest_version: 1` | `manifest_version: 2`，新增 `capabilities`、`i18n` 等字段 |

**数据兼容性**: `custom_commands.json` 数据格式保持不变，可直接迁移使用。

## 📝 注意事项

- **命令优先级**: `添加`、`删除`、`删全局`、`列表` 这几个管理命令的优先级高于您自定义的触发命令。例如，即使你创建了一个名为"列表"的自定义命令，输入 `.列表` 时，也总是会触发"列出所有命令"的功能。
- **图片路径**: 调用回复图片内容时需确保图片放在 `images` 文件夹内（默认路径为 `plugins/custom_commands_plugin/images`），且文件大小不超过 10MB。
- **数据文件**: 所有数据存储在 `custom_commands.json`。插件使用原子写入保护数据安全。请勿手动编辑 `custom_commands.json` 文件，除非你完全清楚其结构和后果。错误的操作可能导致插件无法读取数据。
- **配置格式**: `[settings.group_scopes]` 段下直接写键值对即可（如 `游戏群 = ["123456", "654321"]`），不要在段内再嵌套 `group_scopes = {}`。

Enjoy! 🎉
